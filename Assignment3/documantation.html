API Documentation

Our API is organized around REST. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes and verbs.

1. RETRIEVE THE FULL DATA SET - to list the data of all items:

HTTP Method Endpoint URI Format of Served Resource
GET http://localhost:3000/api/items JSON

The request body does not require any content due to the type of request.

Example of the JSON response sent by the server:

        [{
          "id": 1,
          "product": "Avocado",
          "origin": "Mexico",
          "best_before_date": "January 2021",
          "amount": "210kg",
          "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/f/f2/Persea_americana_fruit_2.JPG/1024px-Persea_americana_fruit_2.jpg"
        },
        {
          "id": 2,
          "product": "Banana",
          "origin": "Indonesia",
          "best_before_date": "March 2021",
          "amount": "300kg",
          "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1920px-Bananas_white_background_DS.jpg"
        }]

Response Code Status

Status code Description
400 Bad Request - The server cannot or will not process the request due to something that is perceived to be a client error.
404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
200 Ok - The request has successfully been executed.

2. CREATE - to add a new product item:

HTTP Method Endpoint URI Format of Served Resource
POST http://localhost:3000/api/items JSON

Values for following item attributes should be provided in JSON form (all are required):

  • "product"
  • "origin"
  • "best_before_date"
  • "amount"
  • "image"

An example of the JSON request sent to the server:

      {
        "product": "Avocado",
        "origin": "Mexico",
        "best_before_date": "January 2021",
        "amount": "230kg",
        "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg"
      }

The request will receive the corresponding status code indicating the outcome of the action.

Response Code Status

Status code Description
400 Bad Request - The server cannot or will not process the request due to something that is perceived to be a client error.
404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
201 Created - The request has been fulfilled and resulted in a new resource being created.

4. UPDATE - to change data of a specific item:

HTTP Method Endpoint URI Format of Served Resource
PUT http://localhost:3000/api/items/ID=? JSON

Values for following item attributes should be provided in JSON form (not all are required):

  • "product"
  • "origin"
  • "best_before_date"
  • "amount"
  • "image"

An example of the JSON request sent to the server:

      {
        "id": "2",
        "product": "Mango",
        "origin": "Sudan",
        "best_before_date": "January 1999",
        "amount": "230kg",
        "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg"
      }

The request will receive the corresponding status code indicating the outcome of the action.

Response Code Status

Status code Description
400 Bad Request - The server cannot or will not process the request due to something that is perceived to be a client error.
404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
204 No Content - The request has succeeded, but returns no message body.
