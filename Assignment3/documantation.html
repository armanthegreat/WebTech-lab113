API Documentation

Our API is organized around REST. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies,returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

Create

The unique identifier that allows the user to add a new items to the database with the pramiters shown below.

curl POST http://localhost:3000/api/item

An example of the request sent to the databse :
{
"id": 2,
"product": "Mango",
"origin": "Sudan",
"best_before_date": "January 1999",
"amount": "230kg",
"image":
"https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg"
}

The response of the server depending on your request :

Status code Description
400 Bad Request - The server cannot or will not process the request due to something that is perceived to be a client error.
404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
201 Created - The request has been fulfilled and resulted in a new resource being created.

Retrieving items

With the help of the this identifier you can retreive every item from the database or just retrieve one item by adding the id of the item to URI

To get every data : curl GET http://localhost:3000/api/items

To get a specific data : curl GET http://localhost:3000/api/items/id=?

An example of the response sent by the database :
{
"id": 1,
"product": "banana",
"origin": "The Netherlands",
"best_before_date": "November 2019",
"amount": "100kg",
"image": "https://upload.wikimedia.org/wikipedia/commons/thumb/e/ee/Apples.jpg/512px-Apples.jpg"
},
{
"id": 2,

"amount": "230kg",
"image": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg"

The responses of the server depending on your request :

Status code Description

404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
200 Ok - The request has successfully been executed.

Update

To be able to update any specification of an item in the database the following identifier would acaccess a product through its specific id and update its fields.

An example of the request sent to the database :
{
"id": "2",
"product": "Mango",
"origin": "Sudan",
"best_before_date": "January 1999",
"amount": "230kg",
"image": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg"
}

The response of the server depending on your request :

Status code Description

404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.

Delete

The following identifier has the ability to reset the entire database, it also could also just delete a single item in the database by adding the items id to the URI.

To reset the whole databse :curl DELETE http://localhost:3000/api/items

To Delete a single item : curl DELETE http://localhost:3000/api/items/id=?

The response of the server depending on your request :

Status code Description
400 Bad Request - The server cannot or will not process the request due to something that is perceived to be a client error.
204 No Content - The request has succeeded but returns no message body.
