API Documentation

Our API is organized around REST. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes and verbs.

CREATE

The unique identifier that allows the user to add a new item to the database with the pramiters shown below.

POST http://localhost:3000/api/item

An example of therequestsent to the server:

      {
        "id": 2,
        "product": "Avocado",
        "origin": "Mexico",
        "best_before_date": "January 2021",
        "amount": "230kg",
        "image":
        "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg"
      }
Status code Description
400 Bad Request - The server cannot or will not process the request due to something that is perceived to be a client error.
404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
201 Created - The request has been fulfilled and resulted in a new resource being created.

READ

With the help of the this identifier you can retreive every item from the database or just retrieve one item by adding the id of the item to URI

To get all items :

GET http://localhost:3000/api/items

To get a specific data with ID:

GET http://localhost:3000/api/items/id=?

DELETE

The following identifier has the ability to reset the entire database, it also could also just delete a single item in the database by adding the items id to the URI.

To reset the whole databse

DELETE http://localhost:3000/api/items

To Delete a single item

DELETE http://localhost:3000/api/items/id=?

Status code Description
400 Bad Request - The server cannot or will not process the request due to something that is perceived to be a client error.
204 No Content - The request has succeeded, but returns no message body.
