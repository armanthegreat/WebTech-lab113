API Documentation

Our API is organized around REST. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies,returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

Create

The unique identifier that allows the user to add a new iteams to the database with the pramiters shown below.

POST http://localhost:3000/api/item

An example of the response sent by the server to the following identifier : { "id": 2, "product": "Mango", "origin": "Sudan", "best_before_date": "January 1999", "amount": "230kg", "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg" } If the identifier fails to add the new item into the database it would respond with a sataus code (400) If the identifier succesfully adds the new item into the database it would respond with a status code (200)

Retrieve All items

With the help of the this identifier you can retreive all the data from the database and view them

GET http://localhost:3000/api/items

An example of the response sent by the server to the this specific identifier : { "id": 1, "product": "banana", "origin": "The Netherlands", "best_before_date": "November 2019", "amount": "100kg", "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/e/ee/Apples.jpg/512px-Apples.jpg" }, { "id": 2, "product": "Mango", "origin": "Sudan", "best_before_date": "January 1999", "amount": "230kg", "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg" }

Retrieve item

To get a specific iteam from the database the following identifier would help to retrieve a single iteam based on its unique id

GET http://localhost:3000/api/item/id=?

An example of the response sent to the server and the response : http://localhost:3000/api/item/1 { "id": 1, "product": "banana", "origin": "The Netherlands", "best_before_date": "November 2019", "amount": "100kg", "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/e/ee/Apples.jpg/512px-Apples.jpg" }

Update

To be able to update an iteam in the database the following identifier would acess a specific id and update its fields

curl PUT http://localhost:3000/api/items

An example of the request sent to the server : { "id": "2", "product": "Mango", "origin": "Sudan", "best_before_date": "January 1999", "amount": "230kg", "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/44/Bananas_white_background_DS.jpg/1280px-Bananas_white_background_DS.jpg" }

Delete

To remove a specific iteam from the databse we need to use the following identifier to be able to do so.

DELETE http://localhost:3000/api/items/id=?

An example of the request sent to the server and the response : curl DELETE http://localhost:3000/api/items/id=1

Reset

To remove every data in the table and reset it we could use the following identifier and empty out the database

DELETE http://localhost:3000/api/items

An example of the response sent to the server and the response : DELETE http://localhost:3000/api/items/id=1

STATUS CODE DESCRIPTION 201 Created - The request has been fulfilled and resulted in a new resource being created. 204 No Content - The request has succeeded but returns no message body. 400 Bad Request - The server cannot or will not process the request due to something that is perceived to be a client error. 404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
